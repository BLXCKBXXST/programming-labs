# Оценка 5 — RLE-сжатие бинарного файла

Вариант 21: **Кроссовки** — сотрудники с полями Имя, ID, Уровень.

## Что делает программа

1. Читает `staff.csv`, выводит таблицу (оценка 3)
2. Сохраняет структуры в `database.dat` через `fwrite()` (оценка 4)
3. Поиск по ID через `fread()` (оценка 4)
4. `compress()` — RLE-сжатие `database.dat` → `database.rle`
5. `decompress()` — восстановление `database.rle` → `database_restored.dat`
6. Отчёт о сжатии в консоль

## Структура файлов

```
grade-5/
├── main.c        # точка входа
├── employee.c    # вся логика: CSV, fwrite, fread, поиск, compress, decompress
├── employee.h    # структура Employee + объявления
├── menu.c        # интерактивное меню
├── menu.h
├── staff.csv
├── Makefile
└── README.md
```

## RLE-формат

Каждая пара байт в `database.rle`:
```
[count][value]   — повторить value count раз
```

Пример: последовательность `00 00 00 41 41` → `03 00 02 41`

## Сборка и запуск

```bash
make
./program staff.csv
```
