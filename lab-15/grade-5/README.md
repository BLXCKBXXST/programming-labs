# Лабораторная работа №15 — grade-5

## Описание

Программа загружает записи из файла `.csv`, `.dat` или `.rle` и предоставляет интерактивное меню для работы с данными. Включает весь функционал оценки 4: сохранение в бинарный `database.dat` через `fwrite()` и поиск по ID через `fread()`.

## Структура проекта

```
grade-5/
├── main.c       — точка входа, определение типа файла
├── record.c     — загрузка, вывод, сохранение, сжатие
├── record.h     — структура Record, константы, прототипы
├── menu.c       — интерактивное меню
├── menu.h       — прототип run_menu
├── Makefile
├── staff.csv    — тестовый CSV (набор 1)
└── staff2.csv   — тестовый CSV (набор 2)
```

## Сборка

make
```

```

## Меню

Меню формируется динамически в зависимости от типа загруженного файла:

| Пункт | CSV | DAT | RLE |
|-------|-----|-----|-----|
| 1. Поиск по ID | ✓ | ✓ | ✓ |
| 2. Показать все записи | ✓ | ✓ | ✓ |
| 3. Сохранить в DAT | ✓ | — | ✓ |
| 4. Сохранить в CSV | — | ✓ | ✓ |
| 5. Сжать в RLE | ✓ | ✓ | — |
| 0. Выход | ✓ | ✓ | ✓ |

> Поиск по ID (пункт 1) работает для всех форматов — данные сначала загружаются в память, поиск ведётся по массиву.

## Формат CSV

Разделитель — `;`, без заголовка:

```
Имя;ИД;Уровень
```

Пример (`staff.csv`):
```
Elon Musk;5001;Level_A
Jeff Bezos;5002;Level_B
James Wilson;6001;Level_C
```

## Формат RLE

RLE-сжатие работает на уровне байт. Каждая последовательность одинаковых байт записывается как два байта:

```
[count: 1 байт][value: 1 байт]
```

Пример: последовательность `0x00 0x00 0x00 0x41` запишется как `03 00 01 41`.

- `compress()` — читает `database.dat`, записывает `database.rle`
- `decompress()` — восстанавливает данные в `database_restored.dat`

После сжатия выводится отчёт:

```
До: 1.23 Кб -> После: 0.89 Кб. Сжатие: 27.6%
```
