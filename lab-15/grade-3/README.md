# Оценка 3 — Чтение текстового CSV-файла — сотрудники с полями Имя, ID, Уровень.

## Что делает программа

1. Принимает имя CSV-файла через аргумент командной строки
2. Читает файл построчно через `fgets()`
3. Разбивает поля по `;` через `strtok()`
4. Выводит отформатированную таблицу в консоль

## Обработка ошибок

- Если аргумент не передан — программа выводит подсказку по использованию: `Usage: ./program <file.csv>`
- Если файл не существует или недоступен — ошибка выводится в `stderr`, программа завершается с кодом `1`
- Строки с неполным набором полей (меньше трёх) пропускаются без ошибки

## Структура файлов

```
grade-3/
├── main.c          # чтение CSV через fgets/strtok, вывод таблицы
├── main_syscall.c  # то же самое через open/read/write/close (★)
├── staff.csv       # входные данные
├── Makefile
└── README.md
```

## Формат staff.csv

```
Elon Musk;5001;Level_A
Jeff Bezos;5002;Level_B
```

## Сборка и запуск

```bash
make              # собирает оба бинарника
./program staff.csv
./program_syscall staff.csv
```

Отдельная сборка:

```bash
make program          # только стандартная версия
make program_syscall  # только версия с syscall
```

## ★ Вариант с системными вызовами (`main_syscall.c`)

Задание «под звёздочкой»: перепиcать без функций стандартной библиотеки ввода-вывода.

| Стандартная библиотека | Системный вызов |
|---|---|
| `fopen()` | `open()` |
| `fgets()` | `read()` побайтово + ручной поиск `\n` |
| `fprintf()` / `printf()` | `write()` в `STDOUT_FILENO` / `STDERR_FILENO` |
| `fclose()` | `close()` |

Поведение и формат вывода идентичны `program`.
