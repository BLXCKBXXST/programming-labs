# Оценка 3 — всё в одном файле

Вариант 21: **Кроссовки** — Бренд, Модель, Размер, Цена.

## Что делает программа

1. Создаёт массив из 10 кроссовок (`#define N 10`)
2. Заполняет его случайными данными
3. Выводит таблицу **до** сортировки
4. Сортирует по цене (пузырьком)
5. Выводит таблицу **после** сортировки

## Структура данных

```c
typedef struct {
    char brand[20];  // бренд
    char model[20];  // модель
    int  size;       // размер (36–44)
    int  price;      // цена ($)
} Sneaker;
```

## Как работает сортировка пузырьком

Представь, что самый дорогой кроссовок — это тяжёлый камень.  
Мы идём по массиву и сравниваем каждую пару соседей.  
Если левый дороже правого — меняем их местами.  
После одного прохода самый дорогой окажется в конце.  
Повторяем N-1 раз — массив отсортирован.

```
Проход 1: [300, 100, 200] → [100, 200, 300]  ← 300 «всплыл» в конец
Проход 2: [100, 200, 300] → без изменений
```

Сложность: **O(n²)** — медленно на больших данных, но просто и понятно.

## Сборка и запуск

```bash
gcc main.c -o sneakers
./sneakers
```
