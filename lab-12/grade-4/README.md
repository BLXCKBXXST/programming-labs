# Оценка 4 — многофайловый проект + статическая библиотека

## Структура файлов

```
grade-4/
├── include/
│   └── sneakers.h      # заголовочный файл: структура + объявления функций
├── src/
│   ├── main.c          # только main()
│   └── sneakers.c      # реализация всех функций
```

## Зачем делить на файлы?

Когда проект растёт, держать всё в одном файле неудобно.  
Поэтому разбиваем:
- **`sneakers.h`** — как меню: перечисляем что есть (структура, названия функций)
- **`sneakers.c`** — как кухня: здесь готовится сам код функций
- **`main.c`** — как зал: только вызываем готовые блюда

## Что такое статическая библиотека (.a)?

Это архив из скомпилированных `.o` файлов.  
При сборке программы код из библиотеки **вшивается прямо в исполняемый файл**.  
Плюс: программа работает сама, без лишних файлов рядом.  
Минус: если библиотека изменится — нужно пересобирать всё.

## Сборка шаг за шагом

### Шаг 1 — компилируем sneakers.c в объектный файл
```bash
gcc -c src/sneakers.c -I include -o sneakers.o
```
> `-c` — только компиляция, без линковки  
> `-I include` — говорим gcc где искать .h файлы

### Шаг 2 — создаём статическую библиотеку
```bash
ar rcs libsneakers.a sneakers.o
```
> `ar` — архиватор, `rcs` — создать/обновить архив

### Шаг 3 — собираем программу, линкуя библиотеку
```bash
gcc src/main.c -I include -L . -lsneakers -o sneakers
```
> `-L .` — ищем библиотеки в текущей папке  
> `-lsneakers` — подключаем libsneakers.a

### Или просто (без библиотеки, напрямую):
```bash
gcc src/main.c src/sneakers.c -I include -o sneakers
./sneakers
```
