## 🎓 Структура каждой лабы

Типовая раскладка:

```
lab-NN/
├── Лабораторная работа №NN.pdf    # оригинальное задание
├── README.md                        # оглавление и описание лабы
├── Makefile или CMakeLists.txt      # сборка (если есть)
├── grade-3/                         # на оценку 3 ★
├── grade-4/                         # на оценку 4 ★★
├── grade-5/                         # на оценку 5 ★★★
└── grade-star/                      # под звёздочку (+ балл)
```

В лабах 0, 2 и 11 деление по оценкам устроено иначе (см. README внутри). Не все лабы содержат все четыре уровня — берётся то, что описано в методичке.

## 🛠 Как собирать

В каждой папке либо `Makefile`, либо `CMakeLists.txt`. Универсально:

```bash
cd lab-NN
make all              # или: cmake -B build && cmake --build build
./<имя-программы>
```

Подробности — в README соответствующей лабы.

## 📌 Соглашения

- **Язык:** C11 (где-то ANSI C). Сборка через `gcc -Wall -Wextra -std=c11`.
- **Стиль:** код пишется максимально просто, без избыточных абстракций — учебный, а не промышленный.
- **Псевдокод из методички** сохраняется буквально (см. `lab-02` — фрагменты идентичны исходным `for (s=1, i=1; i<=n; i++) s = s * i;`).
- **Варианты** выбираются по номеру в группе (25) с остатком от количества алгоритмов.
- **Скриншоты** ожидаемого вывода или чертежей задач (например, область для `lab-01/grade-5`) — лежат рядом с кодом как `.png` и вставлены в README.
