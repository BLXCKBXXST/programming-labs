# Оценка 4 — CMakeLists.txt

## Что такое CMake?

CMake — это **генератор Makefile-ов**.  
Ты пишешь один `CMakeLists.txt`, а CMake сам создаёт нужные Makefile под твою систему.  
Полезно когда проект большой или нужно собирать на Windows/Linux/Mac.

## Структура

```
grade-4/
├── CMakeLists.txt    # скрипт сборки для CMake
├── include/
│   └── sneakers.h
└── src/
    ├── main.c
    └── sneakers.c
```

## Как собрать

```bash
# шаг 1: создаём папку для всех генерируемых файлов (cmake не мусорит в исходниках)
mkdir build && cd build

# шаг 2: конфигурируем проект (читает CMakeLists.txt, генерирует Makefile)
cmake ..

# шаг 3: собираем
make

# запуск
./sneakers_app
```

## Что такое LIB_TYPE и зачем выбирать тип библиотеки?

В `CMakeLists.txt` есть опция:
```cmake
set(LIB_TYPE "STATIC" CACHE STRING "Library type: STATIC or SHARED")
```

Это значит — можно выбрать **как** будет собрана библиотека.  
По умолчанию `STATIC`, но можно передать `-DLIB_TYPE=SHARED` при конфигурации.

| | Статическая (STATIC, `.a`) | Динамическая (SHARED, `.so`) |
|---|---|---|
| Как работает | Код библиотеки **вшивается** прямо в исполняемый файл при сборке | Код библиотеки **остаётся отдельным файлом** и подгружается при запуске |
| Нужен `.so` рядом при запуске? | ❌ Нет | ✅ Да |
| Размер exe-файла | Больше | Меньше |
| Обновить библиотеку без пересборки? | ❌ Нет, надо пересобрать всё | ✅ Да, просто заменить `.so` |
| Когда использовать | Маленький проект, удобно распространять | Большой проект, несколько программ используют одну библиотеку |

### Пример: сборка со статической библиотекой (по умолчанию)
```bash
mkdir build && cd build
cmake ..                      # LIB_TYPE=STATIC по умолчанию
make
```

### Пример: сборка с динамической библиотекой
```bash
mkdir build && cd build
cmake .. -DLIB_TYPE=SHARED    # передаём опцию через -D
make
export LD_LIBRARY_PATH=.      # говорим где искать .so при запуске
./sneakers_app
```

## Что значит -DCMAKE_BUILD_TYPE?

Если спросят на экзамене, отвечаешь так:

| Значение | Что делает |
|---|---|
| `Debug` | Включает отладочную информацию, отключает оптимизации. Используешь при разработке. |
| `Release` | Оптимизирует код для скорости. Используешь когда всё готово и нужна скорость. |

```bash
cmake .. -DCMAKE_BUILD_TYPE=Release
```
