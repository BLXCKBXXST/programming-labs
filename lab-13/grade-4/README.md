# Lab 13 — Grade 4

Содержит CMakeLists.txt + структуру src/ / include/ + выбор типа библиотеки через `-DLIB_TYPE`.

## Сборка
```bash
mkdir build && cd build
cmake -DLIB_TYPE=STATIC ..   # или SHARED
cmake --build .
./sneakers
```

## Опция -DCMAKE_BUILD_TYPE

`-DCMAKE_BUILD_TYPE` — это опция CMake, которая управляет режимом сборки. Она определяет,
какие флаги передаются компилятору и насколько агрессивно оптимизируется код.

| Значение | Флаги компилятора | Назначение |
|------------|-------------------|------------|
| `Debug`    | `-g -O0`          | Разработка и отладка |
| `Release`  | `-O3 -DNDEBUG`    | Продакшная сборка |
| `RelWithDebInfo` | `-O2 -g`   | Оптимизация + отладочные символы |
| `MinSizeRel` | `-Os -DNDEBUG`  | Минимальный размер бинарника |

### Debug
- Флаг `-g` добавляет в бинарник отладочные символы — без них `gdb` не сможет показать строки кода
- Флаг `-O0` отключает оптимизацию — код выполняется точно так, как написан, ничего не переставляется
- Бинарник больше и медленнее, но удобен для отладки

### Release
- Флаг `-O3` включает максимальную оптимизацию: инлайнинг функций, развёртывание циклов, векторизация
- Флаг `-DNDEBUG` отключает все `assert()` — проверки не выполняются в продакшне
- Бинарник меньше и быстрее, но отладка через `gdb` невозможна

### Пример использования
```bash
# Debug сборка
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Debug ..
cmake --build .
gdb ./sneakers   # можно отлаживать

# Release сборка
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release ..
cmake --build .
./sneakers       # быстрая оптимизированная сборка
```
