# Задание 3 — `%c` vs `%d`: символы и их коды

Одна и та же переменная типа `char` или `short` может выводиться **как символ** (`%c`) или **как число** (`%d`). Разница только в форматной строке.

## Код

```c
char ch = 63;
unsigned char uch = 'r';
short j = 'b', k = 99;

printf("%c == %d\n", ch, ch);
printf("%c == %d\n", uch, uch);
printf("%c == %c\n", j, k);
```

## Эталонный вывод

```
? == 63
r == 114
b == c
```

## Почему `?` равен 63, а `99` это `'c'`

Символы в C хранятся как **числа** — их коды в таблице ASCII. Литерал `'r'` это синтаксический сахар для числа `114`. И наоборот: число `63` это символ `?`, `99` — это `c`.

Несколько ключевых ASCII-кодов:

| Код | Символ |
|---|---|
| 32 | пробел |
| 48–57 | `0`–`9` |
| 63 | `?` |
| 65–90 | `A`–`Z` |
| 97 | `a` |
| 98 | `b` |
| 99 | `c` |
| 114 | `r` |

Полную таблицу можно посмотреть командой `man ascii`.

## Эксперименты

Попробуйте сами:

- Заменить `char ch = 63` на `char ch = '?'` — вывод не изменится.
- Поменять `short k = 99` на `short k = 'c'` — то же.
- Прибавить к `ch` число: `ch + 1` напечатает `@` (код 64).
