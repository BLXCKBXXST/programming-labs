# Оценка 4 — многофайловый проект + статическая библиотека

Вариант 21: **Кроссовки** — Бренд, Модель, Размер, Цена.

## Структура файлов

```
grade-4/
├── include/
│   └── list.h       # заголовочный файл: структуры + объявления функций
├── src/
│   ├── main.c       # только main() и навигация по списку
│   └── list.c       # реализация всех функций работы со списком
```

## Зачем делить на файлы?

Когда проект растёт, держать всё в одном файле неудобно.  
Поэтому разбиваем:
- **`list.h`** — как меню: перечисляем что есть (структуры, названия функций)
- **`list.c`** — как кухня: здесь готовится сам код функций
- **`main.c`** — как зал: только вызываем готовые блюда

## Что изменилось по сравнению с оценкой 3

| Что изменилось | Оценка 3 | Оценка 4 |
|---|---|---|
| Хранение данных | `Sneaker data` (значение) | `Sneaker *data` (указатель) |
| Тип списка | Односвязный (только next) | Двусвязный (next + prev) |
| Навигация | Только вывод | Клавиши D/A/6/4 |
| Файлов | 1 | 3 (main.c, list.c, list.h) |

## Зачем `Sneaker *data` вместо `Sneaker data`?

Пункт 4 задания требует хранить в узле **указатель на структуру**, а не саму структуру.  
Узел и данные живут в разных местах памяти:

```
Node → [data*] → [Sneaker: Nike, Pro, 42, $350]
       [next*] → следующий узел
       [prev*] → предыдущий узел
```

Поэтому при освобождении нужно два `free`: сначала `free(node->data)`, потом `free(node)`.

## Сборка шаг за шагом

### Шаг 1 — компилируем list.c в объектный файл
```bash
gcc -c src/list.c -I include -o list.o
```
> `-c` — только компиляция, без линковки  
> `-I include` — говорим gcc где искать .h файлы

### Шаг 2 — создаём статическую библиотеку
```bash
ar rcs liblist.a list.o
```
> `ar` — архиватор, `rcs` — создать/обновить архив

### Шаг 3 — собираем программу
```bash
gcc src/main.c -I include -L . -llist -o sneakers_list
```
> `-L .` — ищем библиотеки в текущей папке  
> `-llist` — подключаем liblist.a

### Или просто (без библиотеки):
```bash
gcc src/main.c src/list.c -I include -o sneakers_list
./sneakers_list
```
