# Оценка 3 — всё в одном файле

Вариант 21: **Кроссовки** — Бренд, Модель, Размер, Цена.

## Что делает программа

1. Создаёт **односвязный список** из `n` узлов
2. Заполняет случайными данными о кроссовках
3. Выводит список на экран
4. Освобождает память

## Структура данных

```c
typedef struct {
    char brand[20];
    char model[20];
    int  size;
    int  price;
} Sneaker;
```

## Как работает односвязный список

Каждый узел знает только про **следующий**:

```
[Nike Pro] → [Adidas Super] → [Puma Ultra] → NULL
```

`NULL` в конце — значит «дальше ничего нет».

### Почему `Node **head` (двойной указатель)?

Функция `PushBack` должна **изменить** переменную `head` в `main`.  
Чтобы функция могла менять чужую переменную, передаём **адрес** — `&head`.  
Внутри пишем `*head = ...`, чтобы изменить то, на что указывает адрес.

### Почему `malloc`?

Заранее не известно, сколько узлов будет. `malloc` выделяет память  
**во время работы программы** ровно столько, сколько нужно.  
После использования память нужно освободить через `free`.

## Сборка и запуск

```bash
gcc main.c -o sneakers_list
./sneakers_list
```

Или с передачей n как аргумента:
```bash
./sneakers_list 5
```
