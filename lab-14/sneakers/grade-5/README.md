## Структура файлов

```

├── include/
│   └── list.h       # заголовочный файл: структуры + объявления функций

│   ├── main.c       # построение структуры и навигация
│   └── list.c       # реализация CreateSneaker, CreateNode, PrintNode
```

## Схема структуры

```
S → [1] → [2] → [3] → NULL      (верхний ряд, next вправо)
     |      |     |
    [1] ← [2] ← [3] ← [4] → NULL  (нижний ряд, next влево)
```

Здесь N=3, K=4. Узел [4] нижнего ряда не имеет пары — `cross = NULL`.

## Что изменилось по сравнению с оценкой 4

| Что изменилось | Оценка 4 | Оценка 5 |
|---|---|---|
| Рядов | 1 | 2 (верхний N + нижний K) |
| Длины | Одинаковые | N и K могут отличаться |
| cross у «лишних» | — | NULL (пункт 7) |
| Навигация | D/A | D/A/W/S/R |

## Ключевые понятия

### Что такое cross?

`cross` — вертикальная связь между рядами.  
`upper->cross = lower` и `lower->cross = upper`.  
Если у узла нет пары (длины разные) — `cross = NULL` — это пункт 7 задания.

### Как двигаться вправо по нижнему ряду?

Нижний `next` указывает влево. Поэтому «вправо» делаем через верхний ряд:
```
cur → cross (подняться) → next (шаг вправо) → cross (опуститься)
```
Если на каком-то шаге `NULL` — конец.

## Сборка шаг за шагом

### Шаг 1 — компилируем list.c
```bash
gcc -c src/list.c -I include -o list.o
```
> `-c` — только компиляция  
> `-I include` — говорим gcc где искать .h файлы

### Шаг 2 — создаём статическую библиотеку
```bash
ar rcs liblist.a list.o
```

### Шаг 3 — собираем программу
```bash
gcc src/main.c -I include -L . -llist -o sneakers_list
```

### Или просто (без библиотеки):
```bash
gcc src/main.c src/list.c -I include -o sneakers_list
./sneakers_list 4 3
```

## Навигация

| Клавиша | Действие |
|---------|----------|
| `D` / `6` | Вправо |
| `A` / `4` | Влево |
| `S` / `2` | Вниз (cross) |
| `W` / `8` | Вверх (cross) |
| `R` | Вернуться к S |
| `Q` | Выход |
