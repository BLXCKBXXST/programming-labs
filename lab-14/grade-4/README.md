# Лабораторная работа №14 — Оценка 4

## Что делает программа

Программа строит **двусвязный список** из `n` узлов с комнатами.  
Каждый узел хранит **указатель** на структуру `room` (выделенную отдельно).  
По списку можно перемещаться с клавиатуры: вперёд и назад.

---

## Отличия от оценки 3

| Что изменилось | Оценка 3 | Оценка 4 |
|---------------|----------|----------|
| Хранение данных | `struct room data` (значение) | `struct room *data` (указатель) |
| Тип списка | Односвязный (только next) | Двусвязный (next + prev) |
| Навигация | Только вывод | Клавиши D/A/6/4 |

---

## Ключевые понятия

### Зачем указатель на структуру в узле?

В пункте 4 задания требуется добавить в узел **указатель на структуру**, а не саму структуру.  
Это значит, что `room` хранится отдельно в памяти:

```
Node → [data*] → [room: Замок, 5, 42, 3]
       [next*] → следующий узел
       [prev*] → предыдущий узел
```

Плюс: можно передавать только указатель (8 байт) вместо копии всей структуры.

### Двусвязный список

Каждый узел знает и про следующий, и про предыдущий:

```
NULL ← [Пещера] ⇄ [Замок] ⇄ [Лес] → NULL
  S → ^
```

Указатель `S` всегда на начало. Навигация:
- `D` / `6` — вправо (next)
- `A` / `4` — влево (prev)
- `Q` — выход

### Почему `free(tmp->data)` перед `free(tmp)`?

Узел и структура `room` — это **две разные области памяти**.  
Если освободить только узел — данные зависнут в памяти (утечка).  
Сначала освобождаем содержимое, потом сам контейнер.
