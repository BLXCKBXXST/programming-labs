# ★ Звёздочка — Makefile с объектниками

## Задача

> «Самостоятельно изучить Makefile. Написать Makefile для сборки программы из объектных файлов, которые получаются путем компиляции исходных файлов с кодом.» — из PDF.

Берём исходники из `grade-4/` и собираем через **двухшаговый** процесс:

1. `main.c` → `main.o` (компиляция)
2. `fraction.c` → `fraction.o`
3. `main.o + fraction.o` → `fraction` (линковка)

## Файл [Makefile](Makefile)

```makefile
$(TARGET): $(OBJS)
	$(CC) $(CFLAGS) $^ -o $@        # линкуем оба .o в бинарник

main.o:     ../grade-4/main.c     ../grade-4/fraction.h
	$(CC) $(CFLAGS) -c $< -o $@      # -c = «только компиляция, без линковки»

fraction.o: ../grade-4/fraction.c ../grade-4/fraction.h
	$(CC) $(CFLAGS) -c $< -o $@
```

## Зачем разделять на компиляцию и линковку

| Без разделения | С разделением (Makefile) |
|---|---|
| Меняем 1 строку в `fraction.c` → перекомпилируется ВСЁ | Меняем `fraction.c` → перекомпилируется только `fraction.o` |
| 5 секунд на каждую сборку | 1 секунда |

`make` смотрит **даты модификации** файлов и пересобирает только то, что устарело. На крупных проектах это десятки минут разницы.

## Команды

```bash
make            # собрать
./fraction      # запустить

make clean      # удалить .o и бинарник
make            # пересобрать с нуля
```

## Установка make (если не было)

```bash
sudo apt update
sudo apt install build-essential -y
```

`build-essential` — мета-пакет, ставит `gcc`, `make`, `libc-dev` и другие основные инструменты разработчика.
