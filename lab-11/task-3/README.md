# Задача 3 — Макрос без скобок

## Симптом

```
$ ./before
z = 11    ← а должно быть 36 (т.к. (5+1)² = 36)
```

## Почему

[before.c](before.c):

```c
#define SQR(x) x * x

int z = SQR(y + 1);    // y = 5
```

Препроцессор тупо подставляет текст. `SQR(y + 1)` превращается в:

```c
y + 1 * y + 1
```

По правилам приоритетов умножение выполняется раньше сложения:

```
y + (1 * y) + 1
= 5 + 5 + 1
= 11
```

## Фикс

[after.c](after.c) — оборачиваем **и каждый аргумент, и весь макрос** в скобки:

```c
#define SQR(x) ((x) * (x))
```

Теперь подстановка даёт:

```c
((y + 1) * (y + 1))   // = 6 * 6 = 36 ✓
```

## Правило

> При написании макросов **в скобки заворачиваем всё**:
> - каждое использование аргумента: `(x)`, `(y)` и т.д.;
> - результат всего макроса целиком — `((...))`.
>
> Это защищает от подставленных выражений (`y + 1`, `a * b`, и пр.) и от внешнего контекста (`SQR(2) / 2` без внешних скобок дал бы `2 * 2 / 2 = 2`, а с обёрткой — `((2)*(2)) / 2 = 2`; правильнее, но без обёртки случаются куда более коварные случаи).

## Альтернатива: `inline`-функция

Лучшая практика в современном C — заменить макрос на `static inline` функцию:

```c
static inline int sqr(int x) { return x * x; }
```

Никаких неожиданностей при подстановке, плюс есть проверка типов. Макросы оправданы только когда нужен generic-код или работа на этапе препроцессора.
