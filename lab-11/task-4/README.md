# Задача 4 — Двойной своп оставляет матрицу как есть

## Симптом

```
$ ./before
1 2 3
4 5 6
7 8 9
```

Матрица не изменилась после `transpose()`. Ожидалось:

```
1 4 7
2 5 8
3 6 9
```

## Почему

[before.c](before.c) — функция:

```c
void transpose(int m[N][N]) {
    for (int i = 0; i < N; i++) {
        for (int j = 0; j < N; j++) {
            swap(&m[i][j], &m[j][i]);
        }
    }
}
```

Цикл проходит **все N² ячеек**. Каждую пару `(i, j)` и `(j, i)` свопает **дважды**:

1. На итерации `i=0, j=1` свопает `m[0][1]` ↔ `m[1][0]`.
2. На итерации `i=1, j=0` свопает `m[1][0]` ↔ `m[0][1]`.

Два свопа подряд = ничего. На диагонали (`i == j`) элемент свопается сам с собой.

## Фикс

[after.c](after.c) — внутренний цикл стартует с `j = i + 1`:

```c
void transpose(int m[N][N]) {
    for (int i = 0; i < N; i++) {
        for (int j = i + 1; j < N; j++) {
            swap(&m[i][j], &m[j][i]);
        }
    }
}
```

Теперь проходим только **верхний треугольник** (без диагонали), каждая пара свопается ровно один раз.

| `i` | `j` старт | пары |
|---|---|---|
| 0 | 1 | (0,1), (0,2) |
| 1 | 2 | (1,2) |
| 2 | 3 | — (цикл не выполняется) |

## Проверка

```bash
gcc after.c -o after && ./after
# 1 4 7
# 2 5 8
# 3 6 9
```

## Правило для алгоритмов с обменом по индексам

Если на каждой итерации мы трогаем пару `(i, j)` и `(j, i)` (или вообще симметричные индексы), внутренний цикл должен идти **строго по одной половине** — обычно с `j = i + 1`. Иначе будет двойная обработка.

Тот же принцип:
- сортировка пузырьком — внутренний цикл с `i+1`;
- сравнение всех пар без повторов — `for (i=0..N) for (j=i+1..N)`.
