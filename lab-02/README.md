## 🔍 Что вычисляет каждый алгоритм

### Алгоритм 8 — факториал

```c
for (s=1, i=1; i<=n; i++) s = s * i;
```

`s` стартует с 1 и множится на 1, 2, …, n. На выходе `s = n!`.

| n | n! |
|---|---|
| 0 | 1 |
| 5 | 120 |
| 10 | 3628800 |

> Замечание: `int` переполняется уже на `13! = 6 227 020 800`. В реализации используется `long long`, хватает до `20! ≈ 2.4·10¹⁸`.

### Алгоритм 9 — максимальная цифра

```c
for (n=a, s=0; n!=0; n=n/10) { k=n%10; if (k>s) s=k; }
```

На каждом шаге отщипываем последнюю цифру `k = n % 10` и обновляем максимум.

| a | max digit |
|---|---|
| 123 | 3 |
| 7019 | 9 |
| 1111 | 1 |

### Алгоритм 10 — разворот цифр

```c
for (n=a, s=0; n!=0; n=n/10) { k=n%10; s=s*10+k; }
```

«Перекладываем» последнюю цифру в начало нового числа.

| a | reverse |
|---|---|
| 12345 | 54321 |
| 100 | 1 |
| 7 | 7 |

> Ведущие нули теряются: `reverse(100) = 001 = 1`. Это особенность алгоритма, не баг.

### Алгоритм 11 — `floor(log₂(n))`

```c
for (k=0, m=1; m <= n; k++, m = m * 2);
printf("%d", k - 1);
```

Удваиваем `m`, пока `m ≤ n`. На выходе `m = 2^k > n`, поэтому `2^(k-1) ≤ n < 2^k`, то есть `k − 1 = floor(log₂ n)`.

| n | floor(log₂ n) |
|---|---|
| 1 | 0 |
| 2 | 1 |
| 8 | 3 |
| 10 | 3 |
| 1000 | 9 |

### Алгоритм 12 — сумма цифр

```c
for (n=a, s=0; n!=0; n=n/10) { k=n%10; s=s+k; }
```

Каждую отщипнутую цифру прибавляем к накопителю.

| a | digit sum |
|---|---|
| 12345 | 15 |
| 100 | 1 |
| 9999 | 36 |

## 📐 Почему именно эти алгоритмы

Согласно методичке, вариант берётся из списка группы. У меня номер 25, а всего алгоритмов 17, поэтому:

```
25 mod 17 = 8   →   на оценку 5 идут алгоритмы 8, 9, 10, 11, 12
```

Алгоритмы из методички даны без названий — их и нужно распознать. В файлах исходники сопровождены комментариями с **оригинальным фрагментом** и **расшифровкой**.
