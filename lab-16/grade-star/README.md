# ★ Звёздочка — Обедающие философы (deadlock avoidance)

## Задача

Пять философов сидят за круглым столом, между ними лежат 5 вилок (общая с каждым соседом). Чтобы есть, философу нужны **обе** вилки — левая и правая. Если не получается — он ждёт. Когда поел — кладёт обе обратно. Цель — съесть все 50 блюд из общего котелка.

Постановка задачи — классическая (Дейкстра, 1965). Главный сюрприз: если все философы берут вилки в одном порядке (например, «сначала правую, потом левую»), они могут одновременно взять каждый по правой и навсегда зависнуть, ожидая левую. Это **дедлок**.

## Файлы

| Файл | Что внутри |
|---|---|
| [din_phil.c](din_phil.c) | Оригинал из ветки `kruffka/C-Programming@2025-2026`. Все философы берут правую → левую вилку. Без аргумента — дедлок. |
| [din_phil_fixed.c](din_phil_fixed.c) | Фикс: чётные берут **левую→правую**, нечётные — **правую→левую**. Циклической зависимости в графе ожидания не возникает. |

## Сборка и запуск

```bash
make all

# С большой вероятностью дедлок — все 5 захватили правую вилку
./din_phil

# То же, но философ #1 спит 1 секунду перед захватом — остальные успевают,
# дедлока обычно нет, но это костыль, а не решение.
./din_phil 1

# Фикс через асимметричный захват — гарантированно не дедлочит.
./din_phil_fixed
```

## Почему дедлок исчезает при асимметричном захвате

Дедлок в исходной версии возникает потому, что **все** философы одновременно держат «правую» вилку и ждут «левую». Это создаёт круговое ожидание:

```
phil 0 → fork 0 (взята) → ждёт fork 1
phil 1 → fork 1 (взята) → ждёт fork 2
phil 2 → fork 2 (взята) → ждёт fork 3
phil 3 → fork 3 (взята) → ждёт fork 4
phil 4 → fork 4 (взята) → ждёт fork 0   ← замыкаем кольцо
```

Все четыре условия Коффмана выполнены: взаимное исключение, удержание с ожиданием, отсутствие принудительного освобождения, **круговая зависимость**.

**Фикс**: ломаем последнее условие. У нечётных философов порядок захвата перевёрнут. Теперь, например, философы 0 и 1 **оба** пытаются захватить вилку 1 **первой**. Кто-то один её получит, второй просто подождёт — и дедлока нет.

| id | чётность | сначала | потом |
|---|---|---|---|
| 0 | even | left  = 1 | right = 0 |
| 1 | odd  | right = 1 | left  = 2 |
| 2 | even | left  = 3 | right = 2 |
| 3 | odd  | right = 3 | left  = 4 |
| 4 | even | left  = 0 | right = 4 |

## Альтернативные решения

1. **Мьютекс-«official»** (semaphore с N−1). Разрешает одновременно «брать вилки» максимум N−1 философу — кому-то всегда придётся ждать на «входе», но в этот момент он уже не держит ни одной вилки, поэтому круг не замкнётся.
2. **Иерархия ресурсов**: каждой вилке присваиваем номер, философ всегда берёт сначала вилку с меньшим номером, потом с большим. То же самое, что асимметрия, но более общее.
3. **`pthread_mutex_trylock`** + откат: если не удалось взять вторую вилку, отпустить первую и попробовать снова. Корректно, но даёт livelock (живой замок), пока не добавить случайные задержки.

В данной лабе выбран **асимметричный захват** как самый простой и наглядный.

## Проверка отсутствия дедлока

```bash
# Полный прогон, ~10–15 секунд, завершится сам
time ./din_phil_fixed > /tmp/p.log
tail -2 /tmp/p.log
# Philosopher X is done eating.
# Philosopher Y is done eating.
echo $?    # 0
```

Под `-fsanitize=thread` (ThreadSanitizer) при запуске фиксированной версии не должно быть сообщений о data race / lock-inversion.
